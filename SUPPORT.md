# Support

Use the right channel for the right problem.

## Documentation

Start with:

- [README.md](./README.md)
- [docs/README.md](./docs/README.md)
- [examples/blog-api/README.md](./examples/blog-api/README.md)

## Bugs

Open a GitHub issue when:

- documented behavior does not match runtime behavior
- generated code fails to compile
- a command crashes or produces invalid output

Include:

- Shaperail version
- operating system
- minimal resource file or reproduction repo
- exact command and error output

## Feature Requests

Open a GitHub issue for concrete, scoped feature proposals. Tie the request to
the resource format, runtime behavior, or developer workflow.

## Security Issues

Follow [SECURITY.md](./SECURITY.md) and do not report vulnerabilities in public
issues.

## Wiki

The GitHub wiki is supplementary. Keep canonical, release-coupled user guides in
the repository `docs/` directory.
