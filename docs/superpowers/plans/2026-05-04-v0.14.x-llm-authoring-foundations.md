# v0.14.x — LLM-Authoring Foundations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the additive slice of the LLM-authoring proposal in a v0.14.x patch release: position-aware diagnostics behind a feature flag, an `explain` extension, per-error doc pages with a CI guard, and 4 reference recipes.

**Architecture:** Three-layer change. (1) `shaperail-codegen` gets a central diagnostic registry, an evolved `Diagnostic` struct with optional `span`/`severity`/`doc_url`, and a saphyr-based parser behind a `saphyr-spans` cargo feature. (2) `shaperail-cli` extends `explain` with validations + OpenAPI fragments + `--format json`. (3) `examples/recipes/` and `docs/errors/` get content; CI gains a doc-page-existence check.

**Tech Stack:** Rust 2021, Cargo workspace, `serde_yaml` (current default parser), `saphyr` (new opt-in parser), `serde_json`, `insta` (snapshot tests), Jekyll (docs site).

**Spec reference:** `docs/superpowers/specs/2026-05-04-llm-authoring-gaps-design.md` — sections §2.2, §2.3 (partial), §3.2, §4 (4 of 8 recipes).

**Out of scope (deferred to later trains):** Typed `Context<I, O>`, `shaperail llm-guide` CLI, sectioned guide, declarative primitives, test scaffold generator, remaining 4 recipes, guide-claim snapshot tests.

```rust
use std::path::PathBuf;
use crate::diagnostics::registry::{lookup, Severity};

#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize)]
pub struct Diagnostic {
    pub code: &'static str,
    pub error: String,
    pub fix: String,
    pub example: String,

    #[serde(skip_serializing_if = "Option::is_none")]
    pub span: Option<Span>,

    pub severity: Severity,

    #[serde(skip_serializing_if = "Option::is_none")]
    pub doc_url: Option<String>,
}

#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize)]
pub struct Span {
    pub file: PathBuf,
    pub line: u32,        // 1-indexed
    pub col: u32,         // 1-indexed (UTF-8 byte column)
    pub end_line: u32,
    pub end_col: u32,
}

impl Diagnostic {
    /// Construct a diagnostic. Severity and doc_url are looked up from the registry.
    /// Panics in debug builds if `code` is not in the registry — keeps the registry
    /// authoritative.
    pub fn error(
        code: &'static str,
        error: impl Into<String>,
        fix: impl Into<String>,
        example: impl Into<String>,
    ) -> Self {
        let entry = lookup(code).unwrap_or_else(|| {
            debug_assert!(false, "diagnostic code {} not in registry", code);
            // Production fallback: emit anyway with default severity.
            return &crate::diagnostics::registry::RegistryEntry {
                code, summary: "", severity: Severity::Error,
            };
        });
        Self {
            code,
            error: error.into(),
            fix: fix.into(),
            example: example.into(),
            span: None,
            severity: entry.severity,
            doc_url: Some(format!("https://shaperail.io/errors/{}.html", code)),
        }
    }

    pub fn with_span(mut self, span: Span) -> Self {
        self.span = Some(span);
        self
    }
}
```

```rust
//! Path-indexed spans. Field paths use dot notation (`schema.email.type`)
//! mirroring how diagnostics name fields in their `error` strings.

use crate::diagnostics::types::Span;
use std::collections::HashMap;
use std::path::PathBuf;

#[derive(Debug, Default, Clone)]
pub struct SpanMap {
    file: PathBuf,
    inner: HashMap<String, (u32, u32, u32, u32)>, // (line, col, end_line, end_col)
}

impl SpanMap {
    pub fn new(file: PathBuf) -> Self {
        Self { file, inner: HashMap::new() }
    }

    pub fn insert(&mut self, path: impl Into<String>, line: u32, col: u32, end_line: u32, end_col: u32) {
        self.inner.insert(path.into(), (line, col, end_line, end_col));
    }

    pub fn lookup(&self, path: &str) -> Option<Span> {
        self.inner.get(path).map(|&(line, col, end_line, end_col)| Span {
            file: self.file.clone(),
            line, col, end_line, end_col,
        })
    }

    pub fn is_empty(&self) -> bool {
        self.inner.is_empty()
    }

    pub fn len(&self) -> usize {
        self.inner.len()
    }
}
```

- [ ] **Step 5: Create the saphyr parser**

Create `shaperail-codegen/src/parser_saphyr.rs`:

```rust
//! Saphyr-based YAML parser. Produces the same ResourceDefinition as
//! the default serde_yaml parser, plus a SpanMap with per-field source
//! positions.
//!
//! Strategy:
//! 1. Parse with saphyr to get a position-annotated AST.
//! 2. Walk the AST to build (a) a serde_yaml::Value for downstream
//!    deserialization and (b) the SpanMap keyed by dotted path.
//! 3. Deserialize the serde_yaml::Value into ResourceDefinition.
//!
//! This keeps ResourceDefinition's serde::Deserialize impl unchanged.

#![cfg(feature = "saphyr-spans")]

use crate::span::SpanMap;
use saphyr::{LoadableYamlNode, Yaml, Marker};
use shaperail_core::ResourceDefinition;
use std::path::PathBuf;

#[derive(Debug, thiserror::Error)]
pub enum SaphyrParseError {
    #[error("saphyr parse error: {0}")]
    Parse(String),
    #[error("expected mapping at document root")]
    NotMapping,
    #[error("downstream deserialization failed: {0}")]
    Deserialize(#[from] serde_yaml::Error),
}

pub fn parse_with_spans(yaml: &str) -> Result<(ResourceDefinition, SpanMap), SaphyrParseError> {
    parse_with_spans_in_file(yaml, PathBuf::new())
}

pub fn parse_with_spans_in_file(
    yaml: &str,
    file: PathBuf,
) -> Result<(ResourceDefinition, SpanMap), SaphyrParseError> {
    let docs = Yaml::load_from_str(yaml).map_err(|e| SaphyrParseError::Parse(e.to_string()))?;
    let doc = docs.into_iter().next().ok_or_else(|| SaphyrParseError::Parse("empty doc".into()))?;

    let mut span_map = SpanMap::new(file);
    let value = walk_yaml(&doc, &mut span_map, "");
    let rd: ResourceDefinition = serde_yaml::from_value(value)?;
    Ok((rd, span_map))
}

fn walk_yaml(node: &Yaml, spans: &mut SpanMap, path: &str) -> serde_yaml::Value {
    // Record the span for `path` based on the node's marker.
    if let Some(marker) = node_marker(node) {
        let line = marker.line() as u32;
        let col = marker.col() as u32;
        spans.insert(path, line, col, line, col); // end positions are best-effort
    }

    match node {
        Yaml::Mapping(map) => {
            let mut out = serde_yaml::Mapping::new();
            for (k, v) in map.iter() {
                let key_str = k.as_str().unwrap_or_default().to_string();
                let child_path = if path.is_empty() { key_str.clone() } else { format!("{}.{}", path, key_str) };
                let key_value = serde_yaml::Value::String(key_str);
                let child = walk_yaml(v, spans, &child_path);
                out.insert(key_value, child);
            }
            serde_yaml::Value::Mapping(out)
        }
        Yaml::Sequence(seq) => {
            let mut out = Vec::with_capacity(seq.len());
            for (i, item) in seq.iter().enumerate() {
                let child_path = format!("{}[{}]", path, i);
                out.push(walk_yaml(item, spans, &child_path));
            }
            serde_yaml::Value::Sequence(out)
        }
        Yaml::String(s) => serde_yaml::Value::String(s.clone()),
        Yaml::Integer(n) => serde_yaml::Value::Number((*n).into()),
        Yaml::Real(s) => s.parse::<f64>()
            .ok()
            .and_then(serde_yaml::Number::from_f64)
            .map(serde_yaml::Value::Number)
            .unwrap_or(serde_yaml::Value::Null),
        Yaml::Boolean(b) => serde_yaml::Value::Bool(*b),
        Yaml::Null => serde_yaml::Value::Null,
        Yaml::Alias(_) | Yaml::BadValue => serde_yaml::Value::Null,
    }
}

/// Extract the source marker from a saphyr node. The exact API depends on the

fn node_marker(_node: &Yaml) -> Option<Marker> {
```

```rust
use crate::span::SpanMap;

/// Identical to `diagnose_resource`, but attaches Span data from the
/// SpanMap by looking up each diagnostic's field path.
pub fn diagnose_resource_with_spans(rd: &ResourceDefinition, spans: &SpanMap) -> Vec<Diagnostic> {
    let mut diags = diagnose_resource(rd);
    for d in &mut diags {
        if let Some(path) = field_path_for(d.code, rd) {
            if let Some(span) = spans.lookup(&path) {
                d.span = Some(span);
            }
        }
    }
    diags
}

/// Map a diagnostic code to the dotted field path it concerns. Codes that
/// inherently span the whole resource (SR001, SR003, SR004) map to "" and
/// pick up the document root span.
fn field_path_for(code: &str, _rd: &ResourceDefinition) -> Option<String> {
    match code {
        "SR001" => Some("resource".into()),
        "SR002" => Some("version".into()),
        "SR003" => Some("schema".into()),
        "SR004" | "SR005" => Some("schema".into()),
        // Codes that name a specific field require richer plumbing; in the
        // first cut, every per-field diagnostic returns None and we'll fill
        // these in when migrating the emission sites field-by-field.
        _ => None,
    }
}
```

```rust
        let req_schema = describe_request_shape(rd, ep);
        if !req_schema.is_empty() {
            println!("    request: {}", req_schema);
        }
        println!("    responses:");
        for (status, body) in describe_response_codes(rd, ep) {
            println!("      {}: {}", status, body);
        }
        if !ep.auth().is_empty() {
            println!("    auth: [{}]", ep.auth().join(", "));
        }
    }
}

/// Return a one-liner of the request body shape: comma-separated field
/// names with required-marker (`*`) suffixes. Empty for endpoints that
/// take no body (list, get, delete).
fn describe_request_shape(_rd: &shaperail_core::ResourceDefinition, ep: &shaperail_core::EndpointSpec) -> String {
    if !ep.action_takes_body() {
        return String::new();
    }
    let inputs = ep.input_fields().unwrap_or_default();
    let parts: Vec<String> = inputs.iter()
        .map(|name| format!("{}", name))
        .collect();
    format!("{{ {} }}", parts.join(", "))
}

/// Return (status, body-summary) for every response the endpoint can
/// produce. Includes 200/201 happy path, 401 if auth is set, 422 if
/// any input field has validation, 404 for get/update/delete.
fn describe_response_codes(_rd: &shaperail_core::ResourceDefinition, ep: &shaperail_core::EndpointSpec) -> Vec<(u16, String)> {
    let mut out = Vec::new();
    let success = if ep.action() == "create" { 201 } else { 200 };
    out.push((success, format!("{}", success_body_for(ep))));
    if !ep.auth().is_empty() {
        out.push((401, "Unauthorized".into()));
        out.push((403, "Forbidden".into()));
    }
    if ep.action_takes_body() {
        out.push((422, "Validation error (SR-coded body)".into()));
    }
    if matches!(ep.action(), "get" | "update" | "delete") {
        out.push((404, "Not Found".into()));
    }
    out
}

fn success_body_for(ep: &shaperail_core::EndpointSpec) -> &'static str {
    match ep.action() {
        "list" => "{ data: [<resource>], pagination: {...} }",
        "get" | "create" | "update" => "<resource>",
        "delete" => "{ ok: true }",
        _ => "<custom>",
    }
}
```
