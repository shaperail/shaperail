# v0.14.0 — Codegen and Controller Fixes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bundle four 0.13.1 bug-report fixes into a single `feat!:` v0.14.0 release: (1) struct fields for `default:`/`generated:` columns flip from `Option<T>` to `T`; (2) `before:` / `after:` accept arrays of hook names; (3) `ControllerContext` exposes `path_params`; (4) generated controller modules become reachable from integration tests via a `pub mod resources` aggregator.

**Architecture:** Five surface changes across three crates plus docs. The codegen Option rule is one `model_field_is_optional` predicate update. The hook chain change adds an untagged `HookList` enum in `shaperail-core` and replaces every `c.before.as_deref()` callsite with iteration. `path_params` adds one `HashMap<String, String>` field to `Context` populated from `actix_web::web::Path`. The `pub mod resources` aggregator is two extra lines in `generate_registry_module`. All four ride a single bundled commit chain on the existing branch `feat/v0.13.1-upgrade-guide-and-drift-check`, then a follow-up branch for the v0.14 release.

**Tech Stack:** Rust 1.85, Cargo workspace (4 crates), `serde` (untagged enums), `actix-web` 4 (Path extractor), `sqlx`, `insta` (snapshot tests), `cargo clippy -- -D warnings`, `cargo fmt`.

**Spec:** `docs/superpowers/specs/2026-05-03-v0.14-codegen-and-controller-fixes-design.md`

fn model_field_is_optional(field: &FieldSchema) -> bool {
    if field.nullable {
        return true;
    }
    !(field.primary
        || field.required
        || field.generated
        || field.default.is_some())
}

fn field_is_required(field: &FieldSchema) -> bool {
    !model_field_is_optional(field)
}
```

The two are now exact inverses. The rule reads: "Option iff `nullable=true` OR (no other source of non-null guarantees)."

/// One or more controller hook names declared under `before:` or `after:`.
///
/// YAML accepts both the scalar form (single hook) and the array form
/// (chain of hooks). Rust callers iterate via `names()` regardless of shape.
///
/// ```yaml
/// controller: { before: validate_org }                      # Single
/// controller: { before: [check_currency, check_org_match] } # Multi
/// ```
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
#[serde(untagged)]
pub enum HookList {
    Single(String),
    Multi(Vec<String>),
}

impl HookList {
    /// Returns the hook names as a slice, regardless of whether the YAML
    /// declared the single or multi form.
    pub fn names(&self) -> &[String] {
        match self {
            HookList::Single(name) => std::slice::from_ref(name),
            HookList::Multi(names) => names.as_slice(),
        }
    }

    /// Returns `true` if any name in the list begins with `wasm:`.
    pub fn has_wasm(&self) -> bool {
        self.names().iter().any(|n| n.starts_with(WASM_HOOK_PREFIX))
    }
}
```

    pub fn has_wasm_before(&self) -> bool {
        self.before.as_ref().is_some_and(|h| h.has_wasm())
    }

    /// Returns `true` if any `after` hook references a WASM plugin.
    pub fn has_wasm_after(&self) -> bool {
        self.after.as_ref().is_some_and(|h| h.has_wasm())
    }

    /// Returns the first WASM `before` hook path (without the `wasm:` prefix).
    /// Useful for legacy callers that assume a single WASM hook per endpoint.
    pub fn wasm_before_path(&self) -> Option<&str> {
        self.before
            .as_ref()
            .and_then(|h| h.names().iter().find(|n| n.starts_with(WASM_HOOK_PREFIX)))
            .map(|s| &s[WASM_HOOK_PREFIX.len()..])

    /// Returns the first WASM `after` hook path (without the `wasm:` prefix).
    pub fn wasm_after_path(&self) -> Option<&str> {
        self.after
            .as_ref()
            .and_then(|h| h.names().iter().find(|n| n.starts_with(WASM_HOOK_PREFIX)))
            .map(|s| &s[WASM_HOOK_PREFIX.len()..])

        self.after.as_ref().map(|h| h.names()).unwrap_or(&[])

```rust
fn collect_controller_hooks(resources: &[ResourceDefinition]) -> Vec<(&str, Vec<&str>)> {
    resources
        .iter()
        .filter_map(|resource| {
            let hooks: Vec<&str> = resource
                .endpoints
                .as_ref()
                .map(|eps| {
                    eps.iter()
                        .filter_map(|(_, ep)| ep.controller.as_ref())
                        .flat_map(|c| {
                            c.before_names()
                                .iter()
                                .chain(c.after_names().iter())
                                .map(String::as_str)
                                .filter(|s| !s.starts_with(shaperail_core::WASM_HOOK_PREFIX))
                                .collect::<Vec<_>>()
                        })
                        .collect()
                })
                .unwrap_or_default();
            if hooks.is_empty() {
                None
            } else {
                Some((resource.resource.as_str(), hooks))
            }
        })
        .collect()
}
```

This now iterates through every name in the chain instead of taking just the first.
