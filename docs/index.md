# Shaperail Docs

Shaperail's canonical user-facing documentation lives in this repository so it
can version with the framework releases.

Start here:

- [Getting Started](./getting-started.md)
- [Resource Guide](./resource-guide.md)
- [Auth and Ownership](./auth-and-ownership.md)
- [Migrations and Schema Changes](./migrations-and-schema-changes.md)
- [Docker Deployment](./docker-deployment.md)
- [Example App](../examples/blog-api/README.md)

Why these docs live here:

- docs stay in sync with code changes and releases
- contributors can review docs in the same pull request as code
- crates.io users can trace documentation to the exact tagged source

The GitHub wiki is still useful, but it should be secondary. Use it for notes
that are not tightly coupled to a specific release, such as FAQs, design notes,
or community-maintained walkthroughs.
