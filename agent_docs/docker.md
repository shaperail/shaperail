# Docker Setup

## Overview
Docker is used at three levels:
1. **Dev** — `docker compose up` gives you Postgres + Redis locally
2. **CI** — framework tests run inside Docker (reproducible, no local Rust needed)
3. **Release** — `shaperail build --docker` produces a scratch-based image ≤ 25 MB for user apps

---

## 1. Dev — docker-compose.yml (in framework root)

```yaml
# docker-compose.yml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: shaperail_dev
      POSTGRES_USER: shaperail
      POSTGRES_PASSWORD: shaperail
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U shaperail"]
      interval: 5s
      timeout: 3s
      retries: 10

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 10

volumes:
  postgres_data:
```

### Usage
```bash
docker compose up -d          # start in background
docker compose down           # stop
docker compose down -v        # stop + delete data
docker compose logs -f        # tail logs
```

### Environment after docker compose up
```bash
export DATABASE_URL="postgresql://shaperail:shaperail@localhost:5432/shaperail_dev"
export REDIS_URL="redis://localhost:6379"
```

## 2. CI — Dockerfile for framework development

```dockerfile
# Dockerfile.dev — builds and tests the framework itself
FROM rust:1.85-slim AS builder

# Install system deps
RUN apt-get update && apt-get install -y \
    pkg-config libssl-dev libpq-dev \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /framework

# Cache dependencies layer
COPY Cargo.toml Cargo.lock ./
COPY shaperail-core/Cargo.toml ./shaperail-core/
COPY shaperail-codegen/Cargo.toml ./shaperail-codegen/
COPY shaperail-runtime/Cargo.toml ./shaperail-runtime/
COPY shaperail-cli/Cargo.toml ./shaperail-cli/

# Dummy source to cache deps
RUN mkdir -p shaperail-core/src shaperail-codegen/src shaperail-runtime/src shaperail-cli/src \
    && echo "fn main() {}" > shaperail-cli/src/main.rs \
    && for crate in shaperail-core shaperail-codegen shaperail-runtime; do echo "" > $crate/src/lib.rs; done \
    && cargo build --workspace \
    && rm -rf shaperail-core/src shaperail-codegen/src shaperail-runtime/src shaperail-cli/src

# Real build
COPY . .
RUN cargo build --workspace --release

# Run tests
FROM builder AS tester
CMD ["cargo", "test", "--workspace"]
```

### docker-compose.ci.yml (for running tests in CI)
```yaml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: shaperail_test
      POSTGRES_USER: shaperail
      POSTGRES_PASSWORD: shaperail

  redis:
    image: redis:7-alpine

  test:
    build:
      context: .
      dockerfile: Dockerfile.dev
      target: tester
    depends_on:
      postgres: { condition: service_healthy }
      redis: { condition: service_healthy }
    environment:
      DATABASE_URL: postgresql://shaperail:shaperail@postgres:5432/shaperail_test
      REDIS_URL: redis://redis:6379
```

## 3. Release — what `shaperail build --docker` generates for user apps

When a user runs `shaperail build --docker` on their app, the framework generates:

```dockerfile
# Generated by: shaperail build --docker
# Multi-stage: builder + minimal scratch image

FROM rust:1.85-slim AS builder
RUN apt-get update && apt-get install -y musl-tools pkg-config ca-certificates \
    && rm -rf /var/lib/apt/lists/*
RUN rustup target add x86_64-unknown-linux-musl
WORKDIR /app
COPY . .
RUN cargo build --release --target x86_64-unknown-linux-musl

FROM scratch
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/ca-certificates.crt
COPY --from=builder /app/target/x86_64-unknown-linux-musl/release/my-app /my-app
USER 10001:10001
EXPOSE 3000
ENTRYPOINT ["/my-app"]
```

Target: final image ≤ 25 MB (PRD requirement).

## Rules for Claude
- The `docker-compose.yml` file lives in the framework root
- The `Dockerfile.dev` lives in the framework root
- Generated Dockerfiles for user apps go to `shaperail-cli/src/templates/Dockerfile.tera`
- All CI tests MUST pass with `docker compose -f docker-compose.ci.yml up --abort-on-container-exit`
- The `shaperail build --docker` command is part of M11 (CLI milestone)
