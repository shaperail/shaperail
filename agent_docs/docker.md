# Docker Setup

## Overview
Docker is used at three levels:
1. **Dev** — `docker compose up` gives you Postgres + Redis locally
2. **CI** — framework tests run inside Docker (reproducible, no local Rust needed)
3. **Release** — `steel build --docker` produces a scratch-based image ≤ 25 MB for user apps

---

## 1. Dev — docker-compose.yml (in framework root)

```yaml
# docker-compose.yml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: steelapi_dev
      POSTGRES_USER: steel
      POSTGRES_PASSWORD: steel
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U steel"]
      interval: 5s
      timeout: 3s
      retries: 10

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 10

volumes:
  postgres_data:
```

### Usage
```bash
docker compose up -d          # start in background
docker compose down           # stop
docker compose down -v        # stop + delete data
docker compose logs -f        # tail logs
```

### Environment after docker compose up
```bash
export DATABASE_URL="postgresql://steel:steel@localhost:5432/steelapi_dev"
export REDIS_URL="redis://localhost:6379"
```

## 2. CI — Dockerfile for framework development

```dockerfile
# Dockerfile.dev — builds and tests the framework itself
FROM rust:1.75-slim AS builder

# Install system deps
RUN apt-get update && apt-get install -y \
    pkg-config libssl-dev libpq-dev \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /framework

# Cache dependencies layer
COPY Cargo.toml Cargo.lock ./
COPY steel-core/Cargo.toml ./steel-core/
COPY steel-codegen/Cargo.toml ./steel-codegen/
COPY steel-runtime/Cargo.toml ./steel-runtime/
COPY steel-cli/Cargo.toml ./steel-cli/

# Dummy source to cache deps
RUN mkdir -p steel-core/src steel-codegen/src steel-runtime/src steel-cli/src \
    && echo "fn main() {}" > steel-cli/src/main.rs \
    && for crate in steel-core steel-codegen steel-runtime; do echo "" > $crate/src/lib.rs; done \
    && cargo build --workspace \
    && rm -rf steel-core/src steel-codegen/src steel-runtime/src steel-cli/src

# Real build
COPY . .
RUN cargo build --workspace --release

# Run tests
FROM builder AS tester
CMD ["cargo", "test", "--workspace"]
```

### docker-compose.ci.yml (for running tests in CI)
```yaml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: steelapi_test
      POSTGRES_USER: steel
      POSTGRES_PASSWORD: steel

  redis:
    image: redis:7-alpine

  test:
    build:
      context: .
      dockerfile: Dockerfile.dev
      target: tester
    depends_on:
      postgres: { condition: service_healthy }
      redis: { condition: service_healthy }
    environment:
      DATABASE_URL: postgresql://steel:steel@postgres:5432/steelapi_test
      REDIS_URL: redis://redis:6379
```

## 3. Release — what `steel build --docker` generates for user apps

When a user runs `steel build --docker` on their app, the framework generates:

```dockerfile
# Generated by: steel build --docker
# Multi-stage: builder + minimal scratch image

FROM rust:1.75-slim AS builder
RUN apt-get update && apt-get install -y pkg-config libssl-dev libpq-dev \
    && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY . .
RUN cargo build --release

FROM debian:bookworm-slim AS runtime
RUN apt-get update && apt-get install -y ca-certificates libpq5 \
    && rm -rf /var/lib/apt/lists/* \
    && useradd -r -s /bin/false appuser
COPY --from=builder /app/target/release/my-app /usr/local/bin/my-app
USER appuser
EXPOSE 3000
HEALTHCHECK --interval=30s --timeout=3s \
    CMD curl -f http://localhost:3000/health || exit 1
CMD ["my-app"]
```

Target: final image ≤ 25 MB (PRD requirement).

## Rules for Claude
- The `docker-compose.yml` file lives in the framework root
- The `Dockerfile.dev` lives in the framework root
- Generated Dockerfiles for user apps go to `steel-cli/src/templates/Dockerfile.tera`
- All CI tests MUST pass with `docker compose -f docker-compose.ci.yml up --abort-on-container-exit`
- The `steel build --docker` command is part of M11 (CLI milestone)
